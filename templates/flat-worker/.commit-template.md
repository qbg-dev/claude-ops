# Worker Commit Message Template

All worker commits MUST follow this format. Use `worker-commit.sh` to auto-fill metadata.

## Format

```
<type>(<scope>): <short description> [<mission-item>]

## What changed
- <bullet point 1>
- <bullet point 2>

## Verification
- [x] bun test — <result summary>
- [x] tsc --noEmit — <result summary>
- [x] Deploy test — <health check result>
- [x] Endpoint verified — <specific curl/test result>
- [x] Screenshots — <path or "N/A">

## Context
- Mission item: <R01, R04, F1, etc.>
- Worker: <worker-name>
- Branch: worker/<worker-name>
- Cycle: <cycle number>

Co-Authored-By: Claude <model> <noreply@anthropic.com>
```

## Types

| Type | When |
|------|------|
| `fix` | Bug fix |
| `feat` | New feature or capability |
| `refactor` | Code restructure without behavior change |
| `test` | Adding or updating tests |
| `docs` | Documentation only |
| `chore` | Config, build, tooling changes |

## Scopes

| Scope | Area |
|-------|------|
| `chatbot` | WeChat bot, tools, LLM pipeline |
| `miniapp` | Staff/resident miniapp |
| `admin` | Admin UI pages |
| `dashboard` | BI dashboards, finance, PM workbench |
| `bi` | SQL library, StarRocks queries |
| `security` | Auth, RBAC, IDOR, Ranger |
| `deploy` | Deploy scripts, nginx, infrastructure |

## Verification Markers

- `[x]` = passed
- `[-]` = skipped (with reason)
- `[!]` = failed but committed anyway (explain why)

Failed verification does NOT block commits -- work should never be lost.
But the marker must honestly reflect the result.

## Examples

```
fix(chatbot): tighten room matching to prevent identity spoofing [R01]
```

```
feat(dashboard): add collection rate trend chart to finance module [F12]
```

```
fix(security): scope miniapp work-order stats to actor's authorized projects [R35]
```
